Desktop Update 2 — Images + LCP only

Scope:
- Desktop image/LCP optimization only.
- No UI/UX redesign.
- No animation changes.
- No SEO/sitemap/form/tracking changes.

Changes made:
1. Added fetchpriority="high" to the desktop hero preload.
2. Kept the hero image eager and high priority.
3. Added explicit sizes to hero portrait image tags.
4. Updated large fallback image src values inside <picture> blocks to use the existing optimized desktop WebP versions.
5. Added missing width/height to small arrow images.
6. Verified all non-Facebook tracking <img> tags now have width and height attributes.
7. Kept below-the-fold images lazy-loaded and decoding async.
8. Applied safe lossless PNG compression to some large PNG fallback assets where possible.

Not changed:
- UI/UX layout
- Desktop design
- Mobile design
- Animations
- Carousel behavior
- Text/content
- SEO metadata
- Sitemap
- Forms
- Meta Pixel
- GA4
- URLs/redirects

Expected performance impact:
- Better desktop LCP priority.
- Less risk of layout shift from missing image dimensions.
- Lower fallback image payload for modern browsers and safer image loading.
