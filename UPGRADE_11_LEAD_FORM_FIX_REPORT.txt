UPGRADE 11 — Diagnostic Form Google Sheets Fix

Analysis:
1. The diagnostic page JavaScript was using /assets/lead-page/reserver-diagnostic.js without a version query.
2. Cloudflare headers cache /assets/* for 1 year with immutable, so production browsers can keep an old broken form JS file even after deploy.
3. The previous submit function used fetch(no-cors) with AbortController timeout. On production/mobile this can abort before Apps Script finishes, causing the UI error.
4. The payload used internal names like fullName/businessSector. This upgrade adds common aliases (name, Name, category, Category, source, Source) for Apps Script compatibility.

Fixes applied:
- Added cache-busting version to the diagnostic JS script URL.
- Replaced submit transport with safer chain: sendBeacon -> fetch no-cors keepalive -> hidden iframe fallback.
- Removed risky AbortController timeout from the form submit path.
- Added compatible field aliases for Google Sheets / Apps Script.

Protected:
- No homepage UI changes.
- No desktop changes.
- No menu changes.
- No SEO/Analytics/sitemap/robots changes.
- Google Analytics ID preserved.
