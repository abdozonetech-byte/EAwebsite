Manual Google Sheet verification:
1. Deploy the website.
2. Open:
   https://www.elboubakry.com/reserver-diagnostic/?lead_debug=1
3. A debug panel appears at the bottom of the page.
4. Click “Send TEST LEAD to Google Sheet”.
5. Open the Google Sheet and search for:
   TEST LEAD - Elboubakry Website
6. Delete the test row after verification.

Important limitation:
- The code can send a test lead request, but only the Google Sheet can confirm that the row was received.
- Because browser no-cors requests to Google Apps Script return opaque responses, the front end cannot read the final Sheet response directly.

If the test lead does not appear:
- Check the Apps Script deployment permissions: Execute as Me, access Anyone.
- Confirm the Web App URL ends with /exec.
- Check Apps Script executions/logs.
- Check Cloudflare CSP console errors.
- Ensure the sheet script supports POST data and appends rows correctly.
